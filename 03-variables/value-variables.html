Understanding Values in JavaScript

In JavaScript, a value is any data that can be stored, manipulated, or used in a program. Values are categorized into two main types:

  • Primitive Values: Immutable data types including:
    • Number: Represents numeric values (e.g., 42, 3.14)
    • String: Represents text (e.g., "Hello")
    • Boolean: Represents true or false
    • Undefined: A variable that has not been assigned a value
    • Null: Represents the absence of a value
    • Symbol: Unique and immutable identifiers
    • BigInt: Represents large integers beyond the Number limit
  • Object Values: Complex data types like objects, arrays, and functions that can hold multiple values or properties.

Primitive values are immutable (cannot be changed once created), while objects are mutable (their properties can be modified).

Variables in JavaScript

A variable is a named storage location for holding a value. JavaScript provides three ways to declare variables:

  • var: Function-scoped, hoisted, and can be redeclared. (Older, less preferred)
  • let: Block-scoped, not hoisted, and can be reassigned.
  • const: Block-scoped, not hoisted, and cannot be reassigned (though object properties can be modified).

Variables also have scope (global, function, or block) and are affected by hoisting (where declarations are moved to the top of their scope, but not initializations).

Type Coercion and Variables

JavaScript performs type coercion, automatically converting values between types during operations (e.g., "5" + 3 becomes "53", but "5" * 3 becomes 15). Understanding coercion is crucial when working with variables.

Variable Scope and Hoisting

Scope determines where a variable is accessible:

  • Global Scope: Variables declared outside functions, accessible everywhere.
  • Function Scope: Variables declared with var inside a function, accessible only within that function.
  • Block Scope: Variables declared with let or const inside curly braces {}, accessible only within that block.

Hoisting: JavaScript moves variable declarations (but not initializations) to the top of their scope. For example, var x is hoisted, but let and const are not fully hoisted, leading to a "Temporal Dead Zone" if accessed before declaration.
