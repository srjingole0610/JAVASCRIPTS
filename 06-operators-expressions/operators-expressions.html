Introduction to Operators and Expressions

In JavaScript, operators are special symbols or keywords used to perform operations on operands (values or variables). An expression is a combination of values, variables, and operators that evaluates to a single value. Operators are categorized by their functionality, and their behavior is influenced by operator precedence and associativity.

Common operator types include:

  • Arithmetic: Perform mathematical operations (e.g., +, -).
  • Comparison: Compare values (e.g., ==, ===).
  • Logical: Combine or invert boolean expressions (e.g., &&, ||).
  • Assignment: Assign values to variables (e.g., =, +=).
  • Other: Includes bitwise, ternary, and special operators like typeof.

Comparison Operators and Type Coercion

Comparison operators compare two values and return a boolean (true or false). JavaScript supports loose equality (==, which performs type coercion) and strict equality (===, which checks value and type).

Type coercion can lead to unexpected results, as JavaScript converts operands to a common type before comparison (e.g., "5" == 5 is true, but "5" === 5 is false).

Logical Operators

Logical operators are used to combine or manipulate boolean values:

  • && (AND): Returns true if both operands are true.
  • || (OR): Returns true if at least one operand is true.
  • ! (NOT): Inverts the boolean value.
Logical operators also support short-circuit evaluation, where the second operand is evaluated only if necessary.

Assignment and Other Operators

Assignment operators assign values to variables (e.g., =, +=). Other operators include:

  • Ternary Operator (condition ? expr1 : expr2): A concise conditional expression.
  • typeof: Returns the type of a value.
  • Bitwise Operators: Operate on binary representations (e.g., &, |).
Operator precedence determines the order of operations in complex expressions (e.g., * has higher precedence than +).
